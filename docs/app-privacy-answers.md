# Appプライバシー回答案

現在の実装前提:

- ログインなし
- Google Mobile Ads SDKによる広告あり
- アプリ内課金なし
- AdMob広告配信のため外部通信あり
- サードパーティ解析SDKなし
- 成績、誤答、ブックマークは端末内保存のみ
- オフライン時はアプリ利用不可

## 回答案

- データを収集しますか: はい
- トラッキングに使用されるデータがありますか: AdMobのパーソナライズ広告、IDFA利用、他社アプリ/サイト横断の広告計測を有効にする場合は「はい」

## 想定されるデータ種別

Google Mobile Ads SDKの利用により、以下が収集または利用される可能性があります。App Store Connect提出前に、AdMob管理画面の設定とGoogleの最新の開示情報に合わせて最終確認してください。

- 識別子: デバイスID
- 使用状況データ: 製品の操作、広告データ
- 診断: クラッシュデータ、パフォーマンスデータ
- 位置情報: おおよその位置情報

本アプリ自体は、氏名、メールアドレス、アカウント、学習履歴のサーバー送信、正確な位置情報取得を実装していません。

## 補足

現実装ではATT許可ダイアログと同意管理画面は未実装です。IDFAを使う広告配信やパーソナライズ広告を明確に有効化する場合は、`NSUserTrackingUsageDescription`、ATT許可フロー、必要に応じてGoogle User Messaging Platformによる同意管理を追加し、Appプライバシー回答も更新してください。
